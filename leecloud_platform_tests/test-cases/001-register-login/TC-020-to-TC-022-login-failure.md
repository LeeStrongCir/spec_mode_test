# 登录失败场景测试用例 (SC-03)

| 字段         | 值                                                     |
| ------------ | ------------------------------------------------------ |
| 来源规范     | `specs/001-register-login/spec.md`                     |
| 关联功能需求 | FR-005, FR-006, FR-011, FR-024                         |
| 优先级       | P1                                                     |
| 作者         | Lee Cloud QA Team                                      |
| 创建日期     | 2026-05-13                                             |

## 概述

本文件涵盖登录失败相关的 3 个测试用例，验证系统在异常登录场景下的正确行为，包括错误凭据处理与账号锁定机制。

### 测试用例关联说明

| 用例编号           | 场景                 | 关联关系                                       |
| ------------------ | -------------------- | ---------------------------------------------- |
| TC-020 ~ TC-021    | 无效凭据登录         | **负向测试对**——分别验证"用户不存在"与"密码错误"两种路径，但系统应返回相同的通用错误提示，不得泄露用户是否存在的信息 |

---

## TC-020: 不存在的用户名 + 任意密码 → 通用错误提示

| 字段           | 值                                                        |
| -------------- | --------------------------------------------------------- |
| 用例编号       | TC-020                                                    |
| 用例名称       | 无效用户名登录返回通用错误                                |
| 优先级         | P1                                                        |
| 自动化状态     | Manual                                                    |
| 关联需求       | FR-005, FR-011                                            |

### 前置条件

| 步骤 | 操作                                             | 预期结果         |
| ---- | ------------------------------------------------ | ---------------- |
| 1    | 系统服务正常运行，访问登录页面                    | 登录页面正常加载 |
| 2    | 确认系统中**不存在**用户名为 `user_not_exist_999` 的账号 | 数据库无此记录   |

### 测试数据准备

| 数据项       | 值                           | 说明                     |
| ------------ | ---------------------------- | ------------------------ |
| 测试用户名   | `user_not_exist_999`          | 确保系统中不存在此用户   |
| 测试密码     | `AnyPassword123!`             | 任意密码均可             |

### 操作步骤

| 步骤 | 操作                                                        | 预期结果                                                              |
| ---- | ----------------------------------------------------------- | --------------------------------------------------------------------- |
| 1    | 在登录页面"用户名"输入框中输入 `user_not_exist_999`          | 输入框显示 `user_not_exist_999`                                       |
| 2    | 在"密码"输入框中输入 `AnyPassword123!`                       | 密码框以掩码形式显示                                                  |
| 3    | 点击"登录"按钮                                               | 系统发起登录请求，页面显示加载状态                                    |
| 4    | 等待服务器响应                                               | 服务器返回 401 错误或等效响应                                         |

### 预期结果

| 编号 | 验证项                                                     | 预期结果                                                                 |
| ---- | ---------------------------------------------------------- | ------------------------------------------------------------------------ |
| ER-1 | 错误提示信息                                               | 页面显示通用错误消息 **"用户名或密码错误"**                              |
| ER-2 | 用户名输入框状态                                           | 用户名输入框**保留**原输入值 `user_not_exist_999`                          |
| ER-3 | 密码输入框状态                                             | 密码输入框**已清空**                                                     |
| ER-4 | 错误提示不得揭示用户不存在                                 | 错误消息必须是通用的，不能出现"用户不存在"等区分性提示                     |

## TC-021: 已存在的用户名 + 错误密码 → 通用错误提示（与 TC-020 相同）

| 字段           | 值                                                        |
| -------------- | --------------------------------------------------------- |
| 用例编号       | TC-021                                                    |
| 用例名称       | 正确用户名错误密码返回通用错误                            |
| 优先级         | P1                                                        |
| 自动化状态     | Manual                                                    |
| 关联需求       | FR-005, FR-011                                            |

### 前置条件

| 步骤 | 操作                                             | 预期结果         |
| ---- | ------------------------------------------------ | ---------------- |
| 1    | 系统服务正常运行，访问登录页面                    | 登录页面正常加载 |
| 2    | 准备一个**已存在的**普通用户账号用于测试           | 账号状态正常、未锁定 |

### 测试数据准备

| 数据项       | 值                           | 说明                                             |
| ------------ | ---------------------------- | ------------------------------------------------ |
| 测试用户名   | `test_user_exist`             | 系统中已注册的有效用户                             |
| 测试密码     | `WrongPassword456!`           | 与该用户实际密码**不同**的任意密码                 |
| 正确密码     | `CorrectP@ss2026`             | `test_user_exist` 的实际密码（用于后续恢复验证）    |

### 操作步骤

| 步骤 | 操作                                                        | 预期结果                                                              |
| ------ | ----------------------------------------------------------- | --------------------------------------------------------------------- |
| 1    | 在登录页面"用户名"输入框中输入 `test_user_exist`              | 输入框显示 `test_user_exist`                                          |
| 2    | 在"密码"输入框中输入 `WrongPassword456!`                      | 密码框以掩码形式显示                                                  |
| 3    | 点击"登录"按钮                                               | 系统发起登录请求，页面显示加载状态                                    |
| 4    | 等待服务器响应                                               | 服务器返回 401 错误或等效响应                                         |

### 预期结果

| 编号 | 验证项                                                     | 预期结果                                                                 |
| ---- | ---------------------------------------------------------- | ------------------------------------------------------------------------ |
| ER-1 | 错误提示信息                                               | 页面显示通用错误消息 **"用户名或密码错误"**（**与 TC-020 完全相同**）    |
| ER-2 | 用户名输入框状态                                           | 用户名输入框**保留**原输入值 `test_user_exist`                             |
| ER-3 | 密码输入框状态                                             | 密码输入框**已清空**                                                     |
| ER-4 | 错误提示不得揭示密码错误                                   | 错误消息必须是通用的，不能出现"密码错误"等区分性提示                       |

### 与 TC-020 的差异验证

| 验证项                       | 说明                                                             |
| ---------------------------- | ---------------------------------------------------------------- |
| TC-020 vs TC-021 错误提示    | 两者返回的错误提示必须**完全一致**（"用户名或密码错误"），不得有任何区分 |
| 字段行为一致性               | 两者的用户名保留、密码清空行为必须**完全一致**                     |

## TC-022: 连续 5 次登录失败 → 第 6 次触发账号临时锁定

| 字段           | 值                                                        |
| -------------- | --------------------------------------------------------- |
| 用例编号       | TC-022                                                    |
| 用例名称       | 连续登录失败触发账号锁定                                   |
| 优先级         | P1                                                        |
| 自动化状态     | Manual                                                    |
| 关联需求       | FR-005, FR-006, FR-024                                    |

### 前置条件

| 步骤 | 操作                                             | 预期结果                          |
| ---- | ------------------------------------------------ | --------------------------------- |
| 1    | 系统服务正常运行，访问登录页面                    | 登录页面正常加载                  |
| 2    | 确认测试账号 `lockout_test_user` 未被锁定         | 该账号当前无锁定状态               |
| 3    | 查询锁计数器：登录失败次数为 0                    | `login_failure_count` = 0         |

### 测试数据准备

| 数据项       | 值                           | 说明                                                        |
| ------------ | ---------------------------- | ----------------------------------------------------------- |
| 测试用户名   | `lockout_test_user`           | 系统中已注册的有效用户                                       |
| 错误密码     | `WrongLockPwd!`              | 与该用户实际密码不同的密码                                   |
| 正确密码     | `CorrectLockPwd2026`         | `lockout_test_user` 的实际密码（用于验证锁定前状态）          |
| 锁定阈值     | 5 次                         | 根据 FR-024 定义                                            |
| 锁定时长     | 15 分钟                      | 根据 FR-024 定义                                            |

### 锁计数器状态验证方法

| 方法     | 操作                                                                 | 预期                                                   |
| -------- | -------------------------------------------------------------------- | ------------------------------------------------------- |
| API 查询 | 调用内部管理 API: `GET /api/admin/lock-status/?username=lockout_test_user` | 返回当前失败次数 `failure_count` 与锁定状态 `is_locked` |
| 数据库查询 | 查询 `user_login_attempts` 表该用户名记录                              | `failure_count` 字段值与实际失败次数一致                |
| UI 提示  | 第 1~5 次失败时页面提示                                                | 每次返回 "用户名或密码错误"，无剩余尝试次数提示          |

### 操作步骤

| 步骤 | 操作                                                        | 预期结果                                                              |
| ---- | ----------------------------------------------------------- | --------------------------------------------------------------------- |
| 1    | 第 1 次：输入 `lockout_test_user` + `WrongLockPwd!`，点击登录 | 返回 "用户名或密码错误"，计数器 +1（当前 = 1）                         |
| 2    | 第 2 次：输入 `lockout_test_user` + `WrongLockPwd!`，点击登录 | 返回 "用户名或密码错误"，计数器 +1（当前 = 2）                         |
| 3    | 第 3 次：输入 `lockout_test_user` + `WrongLockPwd!`，点击登录 | 返回 "用户名或密码错误"，计数器 +1（当前 = 3）                         |
| 4    | 第 4 次：输入 `lockout_test_user` + `WrongLockPwd!`，点击登录 | 返回 "用户名或密码错误"，计数器 +1（当前 = 4）                         |
| 5    | 第 5 次：输入 `lockout_test_user` + `WrongLockPwd!`，点击登录 | 返回 "用户名或密码错误"，计数器 +1（当前 = 5），**尚未锁定**           |
| 6    | 第 6 次：输入 `lockout_test_user` + `WrongLockPwd!`，点击登录 | 返回 **"账号已被临时锁定，请 15 分钟后再试"**                          |
| 7    | 确认账号锁定状态                                             | 账号状态标记为 `locked`，锁定开始时间被记录                            |

### 预期结果

| 编号 | 验证项                                                     | 预期结果                                                                     |
| ---- | ---------------------------------------------------------- | ---------------------------------------------------------------------------- |
| ER-1 | 第 1~5 次失败提示                                          | 均返回通用错误 **"用户名或密码错误"**（与 TC-020/TC-021 相同）                |
| ER-2 | 第 5 次失败时账号状态                                       | 账号**尚未锁定**，仍可继续尝试                                                |
| ER-3 | 第 6 次尝试提示                                            | 返回锁定提示 **"账号已被临时锁定，请 15 分钟后再试"**                          |
| ER-4 | 锁定时长                                                   | 锁定持续时间为 **15 分钟**，期间所有登录尝试均被拒绝                           |
| ER-5 | 用户名和密码字段状态（锁定后）                              | 用户名框保留值，密码框清空（同 TC-020/TC-021 行为）                           |
| ER-6 | 锁定期间尝试正确密码                                        | 即使输入正确密码，仍然返回锁定提示信息                                        |

### 补充验证（锁定恢复，可选）

| 步骤 | 操作                                                       | 预期结果                                       |
| ---- | ---------------------------------------------------------- | ---------------------------------------------- |
| 1    | 等待 15 分钟锁定超时                                        | 锁计数器自动清零，账号状态恢复为正常            |
| 2    | 使用正确密码 `CorrectLockPwd2026` 登录                     | 登录成功，跳转至首页                            |
