# 边缘用例：EC-001 — JWT Cookie 过期自动重定向

> **源规格**: `specs/001-register-login/spec.md`
> **相关功能需求**: FR-001 ~ FR-012
> **优先级**: P1
> **测试类型**: 安全测试 — 会话管理
> **创建日期**: 2026-05-13
> **可追溯性**: spec.md → 边缘情况与异常测试 → EC-001
> **严重程度**: 高

---

## 数据准备

| 数据项 | 值 | 备注 |
|--------|------|------|
| 已存在用户名 | `testuser` | 系统中已注册的有效账户 |
| 已存在密码 | `Test@1234` | 符合强度要求 |
| JWT Cookie 名称 | `access_token` | 登录成功后签发的Cookie名称 |
| 受保护路径 | `/console/dashboard` | 代表性的 `/console/*` 路由 |

---

## 前置条件

- 用户 `testuser` 已通过正常流程登录系统，持有有效的 JWT Cookie
- JWT Cookie 尚未过期或已被手动设置为已过期状态

---

## 操作步骤

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 等待 JWT Cookie 自然过期，或手动修改 Cookie `access_token` 使其过期 | JWT Cookie 变为无效状态 |
| 2 | 在浏览器地址栏输入 `/console/dashboard` 并访问 | 系统检测到 JWT Cookie 已过期 |
| 3 | 观察页面跳转行为 | 页面自动重定向至 `/auth/login` |
| 4 | 检查登录页顶部区域 | 显示"会话已过期，请重新登录"提示消息 |
| 5 | 检查浏览器 Cookie 状态 | 过期的 JWT Cookie 已被清除或标记为无效 |

---

## 验收标准

- [ ] 过期 JWT Cookie 访问任意 `/console/*` 路径均重定向至 `/auth/login`
- [ ] 登录页显示"会话已过期，请重新登录"提示
- [ ] 用户无法在 Cookie 过期状态下访问受保护资源
- [ ] 过期 Cookie 被有效清除
