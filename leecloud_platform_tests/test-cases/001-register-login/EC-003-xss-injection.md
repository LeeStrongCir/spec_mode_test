## 数据准备

### XSS/SQL 注入测试输入

| 编号 | 测试输入 | 注入类型 | 说明 |
|------|----------|----------|------|
| XSS-01 | `<script>alert('xss')</script>` | XSS — HTML script 标签 | 标准 XSS 脚本注入 |
| XSS-02 | `<img src=x onerror=alert(1)>` | XSS — HTML 事件属性 | 事件处理器注入 |
| XSS-03 | `javascript:alert(document.cookie)` | XSS — javascript: 协议 | 伪协议注入 |
| XSS-04 | `'; DROP TABLE users; --` | SQL 注入 | SQL DROP 语句尝试 |
| XSS-05 | `' OR '1'='1` | SQL 注入 | 布尔盲注尝试 |
| XSS-06 | `admin'--` | SQL 注入 | 注释截断尝试 |

## 前置条件

1. 系统已启动，前端可正常访问 `/auth/register` 页面
2. 用户未登录
3. 目标用户名（注入载荷）在系统中不存在（排除重复用户名拦截干扰）
4. 密码和邮箱字段均已准备合法值（密码：`Pass@123`，邮箱：`xss_test@leecloud.com`）

## 操作步骤

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 在用户名输入框中输入 XSS-01：`<script>alert('xss')</script>`，填写合法密码 `Pass@123` 并确认，填写合法邮箱 `xss_test@leecloud.com`，点击"注册"按钮 | 系统拒绝该用户名：在用户名下方显示错误提示（如"用户名包含非法字符"），注册流程被拦截；前端不应执行任何 JavaScript 代码 |
| 2 | 在用户名输入框中输入 XSS-02：`<img src=x onerror=alert(1)>`，其余字段同上，点击"注册"按钮 | 同上：拒绝注册，不触发 `onerror` 事件，无弹窗出现 |
| 3 | 在用户名输入框中输入 XSS-03：`javascript:alert(document.cookie)`，其余字段同上，点击"注册"按钮 | 同上：拒绝注册，不读取或弹出 Cookie 内容 |
| 4 | 在用户名输入框中输入 XSS-04：`'; DROP TABLE users; --`，其余字段同上，点击"注册"按钮 | 系统拒绝该用户名：在用户名下方显示错误提示；数据库 `users` 表未被破坏 |
| 5 | 在用户名输入框中输入 XSS-05：`' OR '1'='1`，其余字段同上，点击"注册"按钮 | 同上：拒绝注册，无 SQL 注入成功迹象 |
| 6 | 在用户名输入框中输入 XSS-06：`admin'--`，其余字段同上，点击"注册"按钮 | 同上：拒绝注册，SQL 注释未生效 |
| 7 | （可选）使用浏览器开发者工具查看网络请求，检查各注入载荷是否被发送至服务端 | 若输入已发送至服务端，服务端应返回 400 Bad Request 或 422 Unprocessable Entity，并在响应体中包含非法字符提示 |

## 验证要点

- **前端层面**：输入校验应在客户端拦截明显非法字符（含 HTML 标签），阻止表单提交
- **后端层面**：即使前端校验被绕过，服务端仍须对输入进行清洗/拒绝，不得将原始内容存入数据库
- **安全属性**：整个过程中浏览器不应执行任何注入的 JavaScript 代码
