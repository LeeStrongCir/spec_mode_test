# 测试用例：TC-010 ~ TC-014 — 登录功能

> **源规格**: `specs/001-register-login/spec.md`
> **相关功能需求**: FR-001 ~ FR-012
> **优先级**: P1
> **测试类型**: 功能测试
> **创建日期**: 2026-05-13
> **可追溯性**: 用户故事 #2 — 输入凭据完成登录

---

## 数据准备

| 数据项 | 值 | 备注 |
|--------|------|------|
| 已存在用户名 | `testuser` | 系统中已注册的有效账户 |
| 已存在密码 | `Test@1234` | 符合强度要求（8-32位，含字母+数字+特殊字符） |
| 已存在邮箱 | `testuser@example.com` | 有效邮箱格式 |
| JWT Cookie 名称 | `access_token` | 登录成功后签发的Cookie名称 |
| JWT Cookie 属性 | `HttpOnly` | 防止 JavaScript 访问 |
| JWT Cookie 属性 | `Secure` | 仅通过 HTTPS 传输 |
| JWT Cookie 属性 | `SameSite=Strict` | 防止跨站请求伪造 |

## TC-012: 正确凭据登录

- **优先级**: P1
- **源**: spec.md → 用户故事 #2 → TC-012
- **相关FR**: FR-003（登录认证）、FR-004（Cookie签发）、FR-005（重定向）

|------|------|----------|

## TC-013: "记住我"持久化登录

- **优先级**: P1
- **源**: spec.md → 用户故事 #2 → TC-013
- **相关FR**: FR-006（记住我功能）、FR-004（Cookie签发）

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 访问 `/auth/login` 页面 | 页面正常加载 |
| 2 | 勾选"记住我"复选框 | 复选框状态变为已勾选 |
| 3 | 输入用户名 `testuser` 和密码 `Test@1234` | 无错误提示 |
| 4 | 点击"登录"按钮 | 登录成功，页面重定向至 `/console` |
| 5 | 检查 JWT Cookie 有效期 | Cookie `access_token` 的 Max-Age/Expires 属性为 30 天（2592000秒） |
| 6 | 关闭浏览器 | 会话结束，Cookie 仍存在 |
| 7 | 重新打开浏览器，访问 `/console` | 用户自动保持登录状态，无需重新输入凭据 |
