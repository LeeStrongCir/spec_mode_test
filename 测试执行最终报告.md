### 3.2 按类别归类结论

**第 1 类: 测试代码与应用实现不匹配** — A, B, C, D, G

- 占全部改动的 60%+
- 根本原因: 测试代码编写时基于规格文档而非实际运行环境
- **教训**: data-testid 选择器、API 字段名、状态码必须从实际运行环境校验，不能仅依赖文档
- **全部已修复**

**第 2 类: 环境与依赖缺失** — J, K, L

- 系统无 pip → Python venv 绕过
- 无浏览器 → playwright install chromium
- Edit 操作失误 → 重复代码块删除
- **全部已修复**

**第 3 类: 并发与时序** — E, H, I

- 注册后重定向延迟 → waitFor + parallel 隔离
- hydrate 竞态 → 增加等待
- 用户名冲突 → 随机生成
- **部分修复** (应用层时序问题需开发跟进)

**第 4 类: 安全/路由机制** — F, M

- CSRF 双因素验证 → 已通过代码适配修复
- 登出后客户端路由绕过 → 需确认是否存在安全
- **部分已修复，1 项待确认**
