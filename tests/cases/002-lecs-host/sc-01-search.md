# SC-01: 搜索导航测试用例

> **场景**: 在控制台全局搜索栏中搜索"LECS主机"或"云服务器"，并跳转至 `/console/lecs-hosts/list`
> **功能模块**: LECS 主机管理
> **编写日期**: 2026-05-12
> **文档版本**: v1.0

---

## 前置条件

| # | 条件 |
|---|------|
| 1 | 测试人员已使用有效账号登录 Lee 云平台控制台 |
| 2 | 控制台顶部全局搜索栏正常显示且可交互 |
| 3 | 搜索服务后端运行正常，索引中包含"LECS主机"条目 |
| 4 | 浏览器为桌面端桌面浏览器，分辨率 ≥ 1280px |

## 数据准备

| # | 数据 | 用途 |
|---|------|------|
| 1 | 搜索引擎已索引"LECS主机"条目（含别名"云服务器"） | TC-001 ~ TC-003 通用 |

> 本场景无需额外数据准备。

---

## TC-001: 验证搜索栏关键词匹配与高亮

| 属性 | 值 |
|------|------|
| **优先级** | P1 |
| **类型** | 手工功能测试（前端 UI） |
| **前置条件** | 用户已登录控制台，搜索栏处于空闲状态 |

### 测试步骤

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 将鼠标移至控制台顶部全局搜索栏输入框，点击使其获得焦点 | 输入框获得焦点，光标闪烁，下拉面板未弹出 |
| 2 | 在搜索框中输入 `LECS`（不区分大小写均应变大） | 输入框显示已键入的文本"LECS" |
| 3 | 等待下拉搜索结果面板出现（最长等待 3 秒） | 下拉面板在 **≤ 1 秒** 内自动展开，包含搜索结果列表 |
| 4 | 检查结果列表中是否存在"LECS主机"条目 | "LECS主机"条目出现在搜索结果列表中 |
| 5 | 观察"LECS主机"条目中的文字高亮状态 | 条目文本中"LECS"片段以高亮样式（背景色或加粗）显示，其余文字"主机"保持正常样式 |

### 验证点

| # | 验证项 | 通过标准 |
|---|--------|----------|
| V1 | 搜索结果响应时间 | 从输入完成到结果出现 ≤ 1 秒 |
| V2 | 关键词匹配准确性 | "LECS主机"条目必须出现在结果中 |
| V3 | 高亮正确性 | 仅"LECS"片段高亮，"主机"不高亮 |
| V4 | 高亮视觉显著性 | 高亮文字与非高亮文字有明显视觉区分（颜色差异或背景色） |

### 通过/失败判定

- **通过**：全部验证点 V1~V4 通过
- **失败**：任一验证点不通过

## TC-002: 验证模糊搜索匹配

| 属性 | 值 |
|------|------|
| **优先级** | P1 |
| **类型** | 手工功能测试（前端 UI） |
| **前置条件** | 用户已登录控制台，搜索栏处于空闲状态 |

### 测试步骤

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 清空搜索框中的全部内容（如有），确认输入框为空 | 搜索框为空，下拉面板收起 |
| 2 | 在搜索框中输入 `云服` | 输入框显示已键入的文本"云服" |
| 3 | 等待下拉搜索结果面板出现（最长等待 3 秒） | 下拉面板在 **≤ 1 秒** 内自动展开 |
| 4 | 检查结果列表中是否存在"LECS主机"条目 | "LECS主机"条目出现在搜索结果列表中 |
| 5 | 观察"LECS主机"条目中的文字高亮状态 | "LECS主机"条目中存在高亮显示的匹配片段（对应输入的"云服"模糊匹配的中文别名映射区域） |

### 验证点

| # | 验证项 | 通过标准 |
|---|--------|----------|
| V1 | 搜索结果响应时间 | 从输入完成到结果出现 ≤ 1 秒 |
| V2 | 模糊匹配准确性 | "LECS主机"条目必须出现在结果中（即使输入的是"云服"而非精确关键词） |
| V3 | 模糊高亮正确性 | 匹配片段有明确高亮标识 |
| V4 | 无多余结果干扰 | 列表中不应出现与"云服"无关的随机条目 |

### 通过/失败判定

- **通过**：全部验证点 V1~V4 通过
- **失败**：任一验证点不通过

## TC-003: 验证点击跳转至列表页

| 属性 | 值 |
|------|------|
| **优先级** | P1 |
| **类型** | 手工功能测试（前端导航） |
| **前置条件** | 用户已登录控制台，搜索栏处于空闲状态 |

### 测试步骤

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 在搜索框中输入 `LECS` | 输入框显示"LECS" |
| 2 | 等待下拉搜索结果面板出现，确认其中包含"LECS主机"条目 | 下拉面板在 **≤ 1 秒** 内展开，"LECS主机"条目可见 |
| 3 | 用鼠标左键点击搜索结果中的"LECS主机"条目 | 点击后下拉面板立即收起 |
| 4 | 观察浏览器地址栏 URL 变化 | 地址栏 URL **精确匹配** `/console/lecs-hosts/list`（无额外查询参数，除非页面默认参数） |
| 5 | 观察页面加载情况（最长等待 5 秒） | LECS 主机列表页完整渲染：包含页面标题、数据表格、操作按钮，无错误提示、无白屏、无控制台红色报错 |
| 6 | （可选）在浏览器开发者工具 Console 面板中检查 | 无 JavaScript 错误（红色 Error 级别日志） |

### 验证点

| # | 验证项 | 通过标准 |
|---|--------|----------|
| V1 | URL 精确匹配 | 浏览器地址栏 URL 为 `/console/lecs-hosts/list` |
| V2 | 页面正常渲染 | 列表页标题栏、数据表格区域、操作按钮均可见 |
| V3 | 无错误渲染 | 页面不显示错误页面（如 404、500、空白页、加载失败提示等） |
| V4 | 前端路由无报错 | 浏览器 Console 中无红色 Error 级别日志 |

### 通过/失败判定

- **通过**：全部验证点 V1~V4 通过
- **失败**：任一验证点不通过
