## TC-020：创建入口与表单页面加载

**优先级**: P1
**关联用户故事**: 用户故事 3 — 创建 LECS 主机

### 前置条件
1. 测试用户已成功登录 Lee 云平台控制台。
2. 当前活跃主机数量 < 100（配额未满）。

| 20-2 | 观察新页面整体布局 | 页面渲染出完整的创建表单，包含以下六大板块标题：**基础配置**、**实例规格**、**操作系统**、**IP 配置**、**购买时长**、**配置费用** |
| 20-3 | 检查"基础配置"板块 | 包含：主机名输入框、用户名输入框、密码输入框 |
| 20-4 | 检查"实例规格"板块 | 包含：计费模式下拉选择（默认"包年/包月"）、实例类型单选（经济型/高性能型）、规格列表选择框 |
| 20-5 | 检查"操作系统"板块 | 包含操作系统镜像下拉选择，选项包含 "Huawei Euler OS"、"Ubuntu"、"Windows" |

| 20-7 | 检查"购买时长"板块 | 包含时长选择控件（1~9 个月、12 个月、24 个月） |

## TC-021：主机名校验规则

**优先级**: P1
**关联用户故事**: 用户故事 3 — 创建 LECS 主机
**校验规则**: 4~10 字符，允许 `[a-zA-Z0-9_]`，不能以 `_` 开头

### 前置条件
1. 已完成 TC-020 的前置条件和步骤。
2. 停留在 `/console/lecs-hosts/create` 页面。

| 步骤编号 | 操作步骤 | 预期结果 |
|----------|----------|----------|
| 21-1 | 在"主机名"输入框中依次输入字符：`_`、`a`、`b`、`c`（即输入 `_abc`） | 输入框下方显示红色错误提示："主机名不能以下划线开头"，输入框边框变为红色 |
| 21-2 | 清空输入框，输入 `ab`（仅2个字符，少于最小长度4） | 输入框下方显示红色错误提示："主机名长度至少为4个字符"，输入框边框变为红色 |
| 21-3 | 清空输入框，输入 `abcdefghijk`（11个字符，超过最大长度10） | 输入框下方显示红色错误提示："主机名长度最多为10个字符"，输入框边框变为红色 |
| 21-4 | 清空输入框，输入 `valid_host1` | 输入框下方**不显示**任何错误提示，输入框边框保持默认颜色，校验通过 |

## TC-022：凭据校验规则

**优先级**: P1
**关联用户故事**: 用户故事 3 — 创建 LECS 主机
**校验规则**:
- 用户名：4~16 字符，允许 `[a-zA-Z0-9!@#$%^&*()-=+]`
- 密码：8~32 字符，需满足复杂度要求

### 前置条件
1. 已完成 TC-020 的前置条件和步骤。
2. 停留在 `/console/lecs-hosts/create` 页面。

| 步骤编号 | 操作步骤 | 预期结果 |
|----------|----------|----------|
| 22-1 | 在"用户名"输入框中输入 `ab`（2个字符，少于最小长度4） | 用户名输入框下方显示红色错误提示："用户名长度至少为4个字符"，输入框边框变为红色 |
| 22-2 | 清空用户名输入框，输入 `valid_user`（合法字符、长度在4~16之间） | 用户名输入框下方**不显示**任何错误提示，输入框边框保持默认颜色，校验通过 |
| 22-3 | 在"密码"输入框中输入 `123`（3个字符，少于最小长度8） | 密码输入框下方显示红色错误提示："密码长度至少为8个字符"，输入框边框变为红色 |
| 22-4 | 清空密码输入框，输入 `Abcdef12!`（9个字符，包含大写、小写、数字、特殊字符，长度在8~32之间） | 密码输入框下方**不显示**任何错误提示，输入框边框保持默认颜色，校验通过 |

## TC-023：实例规格选择与费用实时估算

**优先级**: P1
**关联用户故事**: 用户故事 3 — 创建 LECS 主机

### 前置条件
1. 已完成 TC-020 的前置条件和步骤。
2. 停留在 `/console/lecs-hosts/create` 页面。
3. 表单"基础配置"板块已填入合法的主机名和凭据。

### 前置参考
- 经济型 1vCPU+1GiB 月单价：**100 元/月**
- 经济型 2vCPU+2GiB 月单价：**140 元/月**
- 经济型 2vCPU+4GiB 月单价：**180 元/月**
- 经济型 2vCPU+8GiB 月单价：**240 元/月**
- 高性能型 2vCPU+4GiB 月单价：**160 元/月**
- 高性能型 2vCPU+8GiB 月单价：**200 元/月**
- 高性能型 4vCPU+8GiB 月单价：**260 元/月**
- 高性能型 4vCPU+16GiB 月单价：**500 元/月**

| 步骤编号 | 操作步骤 | 预期结果 |
|----------|----------|----------|
| 23-1 | 确认计费模式为"包年/包月" | "配置费用"区域显示费用估算值，单位标注为 **"元/月"** |
| 23-2 | 选择实例类型为"经济型"，规格选择"1vCPU+1GiB"（100元/月），购买时长选择"3个月" | 配置费用区域显示费用为 **"300 元"**，计算过程：100 × 3 = 300，单位标注 **"元/月"** |
| 23-3 | 保持规格不变，将购买时长切换为"12个月" | 配置费用更新为 **"1200 元"**，计算过程：100 × 12 = 1200，单位标注 **"元/月"** |
| 23-4 | 将计费模式从"包年/包月"切换为"按需计费" | 购买时长选项灰度处理或隐藏；配置费用区域更新显示为 **"3.33 元"**，计算过程：100 ÷ 30 ≈ 3.33，单位标注变更为 **"元/天"** |
| 23-5 | 切换实例类型为"高性能型"，规格选择"4vCPU+16GiB"（500元/月），计费模式"按需计费" | 配置费用区域更新为 **"16.67 元"**，计算过程：500 ÷ 30 ≈ 16.67，单位标注 **"元/天"** |
| 23-6 | 将计费模式切换回"包年/包月"，购买时长选择"1个月" | 配置费用更新为 **"500 元"**，计算过程：500 × 1 = 500，单位标注恢复为 **"元/月"** |
| 23-7 | 验证费用估算随规格切换实时更新 | 在任一计费模式下，切换规格时费用数字立即变化，与所选规格的月单价 × 月数（或 ÷ 30）计算结果一致 |

## TC-024：手工 IP 配置校验

**优先级**: P1
**关联用户故事**: 用户故事 3 — 创建 LECS 主机
**校验规则**:
- 手工配置模式下，IP 地址必须为合法 IPv4 格式
- 掩码数值必须为 8~24 之间的整数

### 前置条件
1. 已完成 TC-020 的前置条件和步骤。
2. 停留在 `/console/lecs-hosts/create` 页面。
3. 表单"基础配置"板块已填入合法的主机名和凭据。

| 步骤编号 | 操作步骤 | 预期结果 |
|----------|----------|----------|
| 24-1 | 在"IP 配置"板块中，将 IP 分配方式从"DHCP 自动分配"切换为"手工配置" | 页面展开显示两个新字段："IP 地址"输入框和"掩码数值"下拉框 |
| 24-2 | 在"IP 地址"输入框中输入 `999.999.999.999` | IP 地址输入框下方显示红色错误提示："请输入有效的 IPv4 地址"，输入框边框变为红色 |
| 24-3 | 清空后输入 `abc.def.ghi.jkl`（非数字格式） | 显示"请输入有效的 IPv4 地址"错误提示 |
| 24-4 | 清空后输入 `192.168.1.100` | IP 地址输入框下方**不显示**任何错误提示，输入框边框保持默认颜色，校验通过 |
| 24-5 | 点击"掩码数值"下拉框，查看选项列表 | 下拉选项包含 8 至 24 之间的全部正整数（即 8, 9, 10, …, 24）共 17 个选项 |
| 24-6 | 从下拉框中选择 `24` | 掩码数值显示为 24，无错误提示 |
| 24-7 | 将 IP 分配方式切换回"DHCP 自动分配" | "IP 地址"输入框和"掩码数值"下拉框收起隐藏，不再参与表单校验 |

## TC-026：配额上限拦截

**优先级**: P1
**关联用户故事**: 用户故事 3 — 创建 LECS 主机
**配额规则**: 每用户最多 **100 台**活跃主机（`deleted_at IS NULL`）

### 前置条件
1. 测试用户已登录且已通过身份验证。
2. 该用户当前已有 **100 台活跃主机**（状态为 正常、已关机、创建失败、关机等，只要 `deleted_at` 为空即计入配额）。

| 步骤编号 | 操作步骤 | 预期结果 |
|----------|----------|----------|
| 26-1 | 验证当前活跃主机数 | 在主机列表页查询，确认该用户拥有 100 台活跃主机 |
| 26-2 | 点击"创建LECS主机"按钮，跳转至 `/console/lecs-hosts/create` | 创建表单页面正常加载（前端不直接禁止进入） |
| 26-3 | 填写完整的合法创建表单数据（参考 TC-025 测试数据） | 表单校验全部通过，"立即购买"按钮可点击 |
| 26-4 | 点击"立即购买"按钮 | **方式 A（前端拦截）**：弹出确认对话框，点击"确定"后请求被前端拦截，显示错误信息："主机数量达到上限"。<br>**方式 B（后端拦截）**：请求发送至后端后收到 409 响应，前端展示错误信息："主机数量达到上限" |
| 26-5 | 尝试在创建表单中不填写完整数据即点击"立即购买" | 表单校验阻止提交，显示对应的字段级错误（与配额拦截相互独立） |
| 26-6 | 观察列表页主机数量 | 主机数量仍然为 100 台，未增加新记录，配额拦截成功 |
