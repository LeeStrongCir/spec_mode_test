# 功能规格说明书：LECS主机

**功能分支**: `002-lecs-host`  
**创建日期**: 2026-05-10  
**状态**: 草稿  
**输入**: 用户描述——"LECS主机（云服务器）功能开发：支持在控制台搜索、列表查看、创建、开关机控制及删除操作，前端页面风格与控制台保持一致"

## 用户场景与测试 *(必填)*

### 用户故事 1 —— 搜索并跳转至LECS主机列表

用户登录Lee云平台控制台后，在顶部搜索栏输入"LECS主机"或"云服务器"。搜索结果智能匹配并显示"LECS主机"条目。点击该条目后，直接跳转至LECS主机列表页。

**设定该优先级的原因**：这是访问所有LECS主机功能的唯一入口。若无可用的导航路径，整个功能将无法使用。

**独立可测**：可通过输入搜索关键词，验证高亮匹配结果，并确认跳转至 `/console/lecs-hosts/list` 来进行完整测试。

**验收场景**：

1. **前提** 用户已登录控制台，**操作** 用户在搜索栏输入"LECS"或"云服"，**预期** 下拉结果中出现"LECS主机"，匹配的关键词高亮显示。
2. **前提** 搜索结果已展示，**操作** 用户点击"LECS主机"条目，**预期** 浏览器跳转至 `/console/lecs-hosts/list`，页面正常加载。

---

### 用户故事 2 —— 查看LECS主机列表与操作矩阵

用户查看云主机的数据表格，表中显示主机名/ID、计费模式、运行状态（彩色标签）、私有IP及操作列。页面标题栏右侧仅有一个"创建LECS主机"按钮——无全局工具栏。操作列提供关机、启动、删除三个按钮。按钮的可操作性严格遵循主机的状态机：不支持的操作根据当前状态置灰。已软删除的主机不在列表中显示。

**设定该优先级的原因**：列表页是资源管理的核心中枢。状态可视化与精确的操作控制是运维安全的基础保障。

**独立可测**：可通过创建处于不同状态（正常、创建中、已关机、创建失败、删除中）的主机，验证状态标签渲染、按钮启用/禁用逻辑及软删除主机的可见性。

**验收场景**：

1. **前提** 列表中已有数据，**操作** 用户查看操作列，**预期** 每一行都显示关机、启动、删除按钮。
2. **前提** 主机状态为"正常" (normal)，**操作** 查看该主机的操作列，**预期** "关机"按钮可点击，"启动"和"删除"按钮置灰。
3. **前提** 主机状态为"已关机" (stopped)，**操作** 查看该主机的操作列，**预期** "启动"和"删除"按钮可点击，"关机"按钮置灰。
4. **前提** 主机状态为"创建失败" (failed)，**操作** 查看该主机的操作列，**预期** 仅"删除"按钮可点击，"关机"和"启动"按钮置灰。
5. **前提** 主机处于"删除中" (deleting) 过渡状态，**操作** 查看列表，**预期** 状态标签显示"删除中"，所有操作按钮均置灰。
6. **前提** 用户删除了一台主机，**操作** 刷新列表页，**预期** 该主机记录不再出现。

### 用户故事 3 —— 创建LECS主机

用户在LECS主机列表页（`/console/lecs-hosts/list`）点击"创建LECS主机"按钮，进入配置表单页面（`/console/lecs-hosts/create`）。表单包含六大配置板块，用户在每个板块中完成选择或输入后，费用估算区域实时更新。所有配置完成后点击"立即购买"弹出确认对话框，汇总所有选择及计算费用。点击"确定"提交创建订单后，用户被重定向到列表页（`/console/lecs-hosts/list`），新主机以"创建中"状态显示，并在30秒内转为"正常"或"创建失败"状态。

**设定该优先级的原因**：资源交付是核心路径。配置复杂度与计费准确性直接影响用户转化率。

**独立可测**：可通过完整走通表单填写流程、验证每步校验规则、确认弹窗内容完整性以及观察列表页异步状态转换来进行完整测试。

**验收场景**：

1. **前提** 用户在列表页，**操作** 用户点击"创建LECS主机"，**预期** 打开创建表单。
2. **前提** 用户填写完表单并点击"立即购买"，**操作** 校验通过，**预期** 弹出确认对话框，展示完整的配置清单与计算费用。点击"确定"通过配额校验后重定向至 `/console/lecs-hosts/list`，新主机显示为"创建中"状态。

#### 表单配置详情

##### 基础配置

- **计费模式**：单选，可选"包年/包月"或"按需计费"，默认选中第一项。
- **主机名**：输入框。支持英文字母、数字、下划线，长度 4-10 个字符，不允许下划线开头。任一规则不满足时在输入框下方显示错误提示。
- **访问凭证**：包含用户名和密码两个输入框。
  - 用户名：支持英文字母、数字、特殊字符，长度 4-16 个字符，不满足时在下方提示错误。
  - 密码：支持英文字母、数字、特殊字符，长度 8-32 个字符，不满足时在下方提示错误。

##### 实例规格

- **实例类型**：单选，可选"经济型"或"高性能型"。
- **具体规格**：必须选择一项，未选时提示错误。

  当选择"经济型"时，可选规格：

  | 规格名称 | vCPU | 内存 | 系统盘 | 月费 |
  |----------|------|------|--------|------|
  | 通用计算机 | 2 | 2 GiB | 40 GB | 100 元 |
  | 通用计算机 | 2 | 4 GiB | 40 GB | 140 元 |
  | 通用计算机 | 2 | 8 GiB | 40 GB | 180 元 |
  | 通用计算机 | 4 | 8 GiB | 40 GB | 240 元 |

  当选择"高性能型"时，可选规格：

  | 规格名称 | vCPU | 内存 | 系统盘 | 月费 |
  |----------|------|------|--------|------|
  | 通用增强计算机 | 2 | 4 GiB | 40 GB | 160 元 |
  | 通用增强计算机 | 2 | 8 GiB | 40 GB | 200 元 |
  | 通用增强计算机 | 4 | 8 GiB | 40 GB | 260 元 |
  | 通用增强计算机 | 8 | 16 GiB | 40 GB | 500 元 |

##### 操作系统

- **镜像**：下拉选择，可选"Huawei Euler OS"、"Ubuntu"、"Windows"，默认选中"Huawei Euler OS"。

##### IP 配置

- **IP 分配方式**：单选，可选"DHCP 分配"或"手工配置"。
- 选择"DHCP 分配"时，无需额外操作。
- 选择"手工配置"时，展开"IP地址"输入框和"掩码数值"下拉框：
  - IP 地址：必须输入合法 IPv4 地址格式，不满足时提示错误。
  - 掩码数值：下拉框，可选 8-24 之间的正整数。

##### 购买时长

- 单选，可选 1-9 个月、1 年、2 年。

##### 配置费用

- 计费模式为"包年/包月"时：费用 = 实例规格月费 × 购买月份，展示单位为"元/月"。
- 计费模式为"按需计费"时：费用 = 实例规格月费 ÷ 30，展示单位为"元/天"。
- 费用随用户选择实时更新。

##### 立即购买

- 点击后弹出确认对话框，展示用户的所有选择结果摘要。
- 对话框提供"取消"和"确定"两个按钮。
  - 点击"取消"：关闭弹窗，保留表单已填内容。
  - 点击"确定"：提交创建请求，通过配额校验后执行。

### 用户故事 4 —— 控制主机生命周期（关机/开机）

用户在LECS主机列表页的操作列中根据主机当前状态点击关机或启动按钮。关机操作使"正常"状态的主机经由"关机中" 过渡到"已关机"。启动操作使"已关机"或"创建失败"状态的主机经由"启动中" 过渡到"正常"。当主机处于过渡状态时，所有操作按钮均置灰，防止并发指令冲突。

**设定该优先级的原因**：用户需要控制主机运行时长的能力以优化成本（关机释放CPU/内存，按需计费主机停计资源费）并从故障中恢复。

**独立可测**：可通过覆盖完整的状态流转链路——正常→关机中→已关机→启动中→正常，配合异步耗时与防冲突逻辑进行完整测试。

**验收场景**：

1. **前提** 主机状态为"正常" (normal)，**操作** 用户点击关机，**预期** 按钮立即置灰，状态变为"关机中"，约10秒后状态变为"已关机"，启动按钮恢复可用。
2. **前提** 主机状态为"已关机" (stopped)，**操作** 用户点击启动，**预期** 按钮立即置灰，状态变为"启动中"，约10秒后状态变为"正常"，关机按钮恢复可用。
3. **前提** 主机处于"关机中" (shutting_down) 过渡状态，**操作** 用户尝试点击关机或启动，**预期** 两个按钮均保持置灰，后端拒绝重复请求。

### 用户故事 5 —— 安全删除LECS主机

在LECS主机列表页，用户仅可对处于"已关机"或"创建失败"状态的主机执行删除操作。点击删除按钮后弹出二次确认对话框。确认后，主机经由"删除中" 状态进入软删除——从列表视图中移除但数据库保留记录（设置 `deleted_at`），配额计数减1。

**设定该优先级的原因**：完成主机生命周期的闭环。禁止运行态主机被删除可防止数据丢失与资源泄漏；异步删除确保流畅的用户界面体验。

**独立可测**：可通过验证运行态主机的删除拦截、已关机态主机的异步删除流程以及配额计数减少来进行完整测试。

**验收场景**：

1. **前提** 主机处于"正常"、"创建中"或"关机中"状态，**操作** 查看删除按钮，**预期** 按钮置灰，或点击时提示"**请先将主机关机，再执行删除操作**"并阻止提交。
2. **前提** 主机处于"已关机"或"创建失败"状态，**操作** 用户点击删除并在弹窗中确认，**预期** 状态立即变为"删除中"，所有操作按钮置灰，3-5秒后该行消失，配额计数减1。

### 用户故事 6 —— 通过 API 管理 LECS 主机（面向其他服务/平台）

外部服务或平台（如计费系统、运维编排平台、第三方集成）需要以编程方式管理 LECS 主机。系统必须提供一组 RESTful API 接口，覆盖主机的列表查询、创建、开关机控制和删除操作，供自动化流程和其他服务调用。

**设定该优先级的原因**：LECS 主机的能力不应仅限于控制台手动操作。自动化运维流程、第三方平台集成、内部服务编排都需要通过 API 复用相同的主机管理能力——缺少 API 将迫使外部系统进行重复开发或绕过规范流程直接操作底层资源，引入安全与一致性风险。

**独立可测**：可通过直接调用 API 端点，验证认证鉴权、请求参数校验、响应格式、状态流转逻辑及并发约束。

**验收场景**：

1. **前提** 调用方持有有效的认证凭证，**操作** 调用 `GET /api/v1/lecs-hosts` 查询列表，支持分页和状态过滤，**预期** 返回当前用户（或管理员视图下全部）的活跃主机列表，响应含分页元数据。
2. **前提** 调用方通过配额校验，**操作** 调用 `POST /api/v1/lecs-hosts` 提交创建请求，**预期** 返回新主机信息及"创建中"状态，后台异步执行。
3. **前提** 主机处于"正常"状态，**操作** 调用 `POST /api/v1/lecs-hosts/{id}/stop`，**预期** 返回接受状态，主机进入"关机中"异步流转。
4. **前提** 主机处于"已关机"或"创建失败"状态，**操作** 调用 `POST /api/v1/lecs-hosts/{id}/start`，**预期** 返回接受状态，主机进入"启动中"异步流转。
5. **前提** 主机处于"已关机"或"创建失败"状态，**操作** 调用 `DELETE /api/v1/lecs-hosts/{id}`，**预期** 返回接受状态，主机进入"删除中"异步软删除。
6. **前提** 主机处于"正常"等不可删除状态，**操作** 调用 `DELETE /api/v1/lecs-hosts/{id}`，**预期** 拒绝请求，返回 403 错误及"请先将主机关机，再执行删除操作"提示。
7. **前提** 调用方无有效认证，**操作** 调用任意 API 端点，**预期** 返回 401 未授权错误。
8. **前提** 普通用户调用 API 查询列表，**操作** 请求中不包含其他用户的过滤条件，**预期** 仅返回该用户自身拥有的主机——越权访问被拒绝。

### 边界场景与异常处理

- **配额上限**：当用户持有的主机总数（所有 `deleted_at IS NULL` 的记录，含正常、关机中、已关机、创建失败、删除中状态）达到 **100台** 时，创建请求被阻断，提示"主机数量达到上限"。
- **创建超时**：若后台创建任务超过60秒未完成，主机状态强制降级为"创建失败"，前端停止轮询，防止无限卡在"创建中"循环。
- **并发锁**：当主机行正在执行异步操作（关机中、删除中等），操作列按钮严格处于禁用状态，阻止一切点击。
- **网络断连**：网络断连期间产生的操作在网络恢复后触发列表页全量刷新，确保与数据库状态一致。
- **表单未填校验**：实例规格未选、IP 格式不正确时，"立即购买"不可提交，对应字段下方展示错误提示。

## 需求 *(必填)*

### 功能需求

- **FR-001**：系统 **必须** 允许用户在控制台搜索栏搜索"LECS主机"并通过搜索结果跳转至LECS主机列表页。
- **FR-002**：系统 **必须** 展示活跃LECS主机的分页列表，每条主机展示主机名、ID、计费模式、状态、私有IP及操作按钮。
- **FR-003**：系统 **必须** 强制基于状态机的操作矩阵，关机和启动按钮的启用/置灰严格依据主机当前状态。
- **FR-004**：系统 **必须** 允许已认证用户通过配置计费模式、主机名、访问凭据、实例规格、操作系统镜像、网络IP模式及购买时长来创建LECS主机。
- **FR-005**：系统 **必须** 在创建表单中实时校验主机名格式（支持字母、数字、下划线，4-10个字符，不以 `_` 开头）。
- **FR-006**：系统 **必须** 在创建表单中实时校验凭据复杂度（用户名4-16个字符，密码8-32个字符）。
- **FR-007**：系统 **必须** 根据所选实例规格和计费模式（包年/包月 = 单价×月数，按需 = 单价÷30/天）实时展示费用估算。
- **FR-008**：系统 **必须** 在提交新主机创建前弹出确认对话框，展示所有配置选项（计费模式、主机名、凭据、实例类型及规格、操作系统镜像、IP 配置、购买时长）及计算费用。对话框提供"取消"和"确定"按钮，点击"取消"关闭弹窗保留表单，点击"确定"提交创建请求。
- **FR-009**：系统 **必须** 强制每位用户最多100台活跃LECS主机的配额（统计所有未软删除的记录），达到上限时阻止创建。
- **FR-010**：系统 **必须** 异步处理关机请求，在约10秒内将主机从"正常"→"关机中"→"已关机"。
- **FR-011**：系统 **必须** 异步处理启动请求，在约10秒内将主机从"已关机"或"创建失败"→"启动中"→"正常"。
- **FR-012**：系统 **必须** 仅在主机处于"已关机"或"创建失败"状态时允许删除，拒绝"正常"、"创建中"及其他过渡状态的删除请求。
- **FR-013**：系统 **必须** 异步处理删除操作，将主机经由"删除中"状态（2-5秒）后标记为软删除。
- **FR-014**：系统 **必须** 在列表视图中隐藏软删除的主机，同时在数据库中保留其记录。
- **FR-015**：系统 **必须** 当列表中存在过渡态主机时，每隔3秒自动轮询主机列表，并在所有主机进入终态后停止轮询。
- **FR-016**：系统 **必须** 将卡死在"创建中"状态超过60秒的主机降级为"创建失败"，并停止对其的轮询。
- **FR-017**：系统 **必须** 强制基于角色的访问控制——管理员管理所有主机，普通用户仅管理自身拥有的主机。
- **FR-018**：系统 **必须** 对所有主机操作（创建、删除、启动、关机）记录审计日志，包括操作人身份、时间戳、IP地址及操作详情。
- **FR-019**：系统 **必须** 使用基于JWT Cookie 的认证方式 验证请求，将未认证用户重定向至登录页（`/auth/login`）。
- **FR-020**：系统 **必须** 对 IP 配置、密码哈希等敏感数据进行加密存储，并对敏感字段应用日志脱敏。
- **FR-021**：系统 **必须** 提供 `GET /api/v1/lecs-hosts` 接口，供外部服务查询主机列表，支持分页和按状态过滤，返回主机核心字段集合。
- **FR-022**：系统 **必须** 提供 `POST /api/v1/lecs-hosts` 接口，供外部服务提交主机创建请求。请求体须包含：计费模式（包年/包月 或 按需计费）、主机名、访问凭证（用户名和密码）、实例类型（经济型或高性能型）、具体规格、操作系统镜像（Huawei Euler OS/Ubuntu/Windows）、网络IP配置（DHCP 或 手工 IP+掩码）及购买时长。返回新主机信息及初始状态。
- **FR-023**：系统 **必须** 提供 `POST /api/v1/lecs-hosts/{id}/stop` 接口，供外部服务触发主机关机操作，请求被异步处理，立即返回接受状态。
- **FR-024**：系统 **必须** 提供 `POST /api/v1/lecs-hosts/{id}/start` 接口，供外部服务触发主机启动操作，请求被异步处理，立即返回接受状态。
- **FR-025**：系统 **必须** 提供 `DELETE /api/v1/lecs-hosts/{id}` 接口，供外部服务触发主机删除操作，仅对"已关机"或"创建失败"状态生效，异步执行软删除。
- **FR-026**：系统 **必须** 对所有 API 接口强制执行与前端一致的认证与鉴权（JWT Cookie 或 Service Token），越权请求返回 401 或 403。
- **FR-027**：系统 **必须** 对 API 接口的请求参数进行校验，不合法参数返回 400 错误及具体字段级别的错误信息。
- **FR-028**：系统 **必须** 为 API 接口提供统一 JSON 响应格式（含成功/失败标识、数据负载、错误码与错误描述），确保调用方一致解析。

### 关键实体

- **LECS 主机**：用户拥有的虚拟计算实例。属性包括主机名、计费模式、实例规格、操作系统镜像、网络配置（DHCP 或 手工 IP/掩码）、状态（创建中 (creating)、正常 (normal)、创建失败 (failed)、关机中 (shutting_down)、已关机 (stopped)、启动中 (starting)、删除中 (deleting)、已删除 (deleted)）、购买时长、费用快照及软删除时间戳。
- **实例规格**：预定义的计算配置方案，分"经济型"和"高性能型"两类。
  - 经济型：2vCPU+2GiB(100元/月)、2vCPU+4GiB(140元/月)、2vCPU+8GiB(180元/月)、4vCPU+8GiB(240元/月)，均配 40GB 系统盘。
  - 高性能型：2vCPU+4GiB(160元/月)、2vCPU+8GiB(200元/月)、4vCPU+8GiB(260元/月)、8vCPU+16GiB(500元/月)，均配 40GB 系统盘。
- **用户配额**：每位用户最多100台活跃 LECS 主机的限制（排除软删除记录），在创建时强制执行。

## 假设

- 用户具备完成异步轮询和实时状态更新所需的稳定网络连接。
- 复用现有的Lee云平台认证系统（基于JWT Cookie）和CSRF保护机制，不做修改。
- 主机计费计算由后端执行，前端仅展示估算值。
- 存在可用的异步后台任务基础设施（如 Celery/RQ）用于处理长时间运行操作（30秒创建、10秒关机/启动、5秒删除）。
- 实例规格及操作系统镜像数据由内部数据源提供；v1版本不包含外部市场集成。
- 按需计费模式下主机停机时，计算资源（CPU/内存）释放且停止资源计费，但存储资源（磁盘/镜像）继续产生费用。
- 移动端和平板端的响应式设计不在v1范围内，仅支持桌面端（1280px 及以上）。
- 主机重启（关机+启动组合）不在v1范围内；用户需分步执行这两个操作。
- 操作期间的网络连接丢失将通过重连后自动刷新 优雅处理。
- API 接口版本采用 `/api/v1/` 前缀，后续不兼容变更通过版本号升级，不影响现有调用方。
- 外部服务（非浏览器端）调用 API 时，支持除 JWT Cookie 之外的 Service Token 认证方式（具体格式由认证系统提供）。
